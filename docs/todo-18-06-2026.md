## 20 features that go beyond world class

### Execution intelligence — the platform gets smarter from use

**1. Win/loss feedback loop**
When an RFP is won or lost, the outcome feeds back into `rfp_win_strategy_builder` as a training signal. Win themes that correlate with wins get weighted up in future strategy generation. Thuon improves with every bid.
- *Entry point*: `rfp_tracker.py` status transitions → new `feedback_store.py` → win_strategy prompt weighting

**2. Proposal win probability model**
Learned predictor (trained on Thuon's own bid history): bid score, issuer, evaluation criteria weights, competitor presence → estimated P(win). Displayed at the human checkpoint before committing to full proposal effort.
- *Entry point*: `rfp_bid_evaluator` output + historical `result_store.py` data → sklearn/lightweight model

**3. Intelligent human-checkpoint learning**
Fixed gates → learned gates. Track which checkpoint decisions led to wins/losses. Bids where `bid_score > 87` and no disqualifiers auto-advance; bids 40–87 escalate to human; below 40 auto-reject. Thresholds update from outcomes.
- *Entry point*: `pipeline_runner.py` human-gate logic + outcome feedback loop from #1

**4. Capability A/B testing framework**
Two versions of a prompt/model compete on the same input. A judge capability scores outputs. Winner accumulates wins; after N rounds it auto-promotes to production. Evolutionary improvement without touching code.
- *Entry point*: new `capabilities/capability_judge.py` + versioned SKILL.md body field + promotion CLI

### Memory and knowledge — institutional knowledge at machine speed

**5. Episodic semantic memory retrieval**
When writing a new proposal section, automatically surface the 3 most relevant past sections from prior wins — vector similarity with temporal decay (recent wins weighted higher), not BM25 keyword match. Zero-shot draft from institutional memory.
- *Entry point*: `memory_store.py` (Chroma already wired) + hook in `rfp_section_writer.py`

**6. Document lineage graph**
Every paragraph in every generated document traces back to: source KB file + line, capability version, model, human edit timestamp. Full provenance for audit, reuse, and debugging. When a past-performance write-up changes, flag every active proposal that cited it.
- *Entry point*: new `core/lineage_store.py`; capabilities annotate output with source refs

**7. Federated knowledge contribution**
Team members submit updates to `data/company/*.md` via a review workflow. Each contribution is attributed, diff-reviewed, quality-scored by AI (consistency with existing claims), then merged. Knowledge doesn't go stale or silently contradict itself.
- *Entry point*: new Flask routes `/settings/company/contribute` + `capabilities/knowledge_reviewer.py`

**8. Competitive intelligence knowledge graph**
Not one-shot competitor analysis but a living graph of competitors, their past wins, certifications, pricing signals, teaming relationships. Each `rfp_competitor_analyst` run upserts into the graph. Query: "who beat us in the last 3 NAICS 541511 awards?"
- *Entry point*: networkx or SQLite adjacency tables in `core/competitor_graph.py`; upsert hook in `rfp_competitor_analyst.py`

### Pipeline architecture — beyond static YAML

**9. Adaptive pipeline compiler**
Instead of hand-authored YAML pipelines, an AI planner inspects the task (RFP complexity, page limit, evaluation criteria) and dynamically selects and sequences capabilities. Short RFP → 5-step pipeline. Full source-selection → 18-step pipeline with parallel section writers.
- *Entry point*: new `capabilities/pipeline_planner.py` + `pipeline_runner.py` dynamic-steps mode

**10. Async pipeline execution with checkpointing**
Pipelines write state after every step to a durable store. If step 11 fails at 2 AM, the scheduler retries from step 10 without re-running earlier steps. Survives model timeouts, crashes, and rate limits.
- *Entry point*: `pipeline_runner.py` + new `core/pipeline_checkpoint_store.py` (SQLite-backed)

**11. Proposal red team mode**
A parallel pipeline where a separate agent plays the evaluator: scores the proposal against published evaluation criteria, attacks each section for weaknesses, ghosts competitors. Output is a structured evaluator score card before submission. Adversarial scoring, not consistency checking.
- *Entry point*: new `capabilities/proposal_red_team.py` + pipeline step after `rfp_consistency_checker`

### Integrations and inputs — more surface area

**12. Multi-modal RFP intake**
Use `minicpm-v` (already configured at `ollama.vision_model`) to read charts, org charts, diagrams, and tables embedded in PDF RFPs. Section L requirements buried in a decision matrix get extracted. Currently text-only.
- *Entry point*: `rfp_ingester.py` + `tools/pdf_extractor.py` → vision model pass for non-text pages

**13. Bid calendar from contract end-date prediction**
Scrape USASpending.gov and SAM.gov for active contracts in your NAICS codes. Infer recompete dates from period-of-performance end dates. Populate a 12-month forward pipeline of likely solicitations before they're posted. Proactive BD vs reactive.
- *Entry point*: new `capabilities/bid_calendar_builder.py` + USASpending API + `calendar_store.py`

**14. Real-time market signal injection**
At capability execution time, pull live data into context: Federal Register notices, agency budget news, industry price indices. RFP responses cite current numbers, not months-old training data. `fx_rates.py` and `news_search.py` tools already exist.
- *Entry point*: new `core/market_signal_provider.py` injected as dep into relevant capabilities

**15. Multi-model routing via LiteLLM**
Consolidate `ai_engine.py` (currently all stubs) as a real router: fast cheap model (qwen) for first-pass research and outlines, frontier model (Claude via LiteLLM at `62.169.25.77`) for final document polish and win strategy. Cost-aware routing with per-capability model config in SKILL.md.
- *Entry point*: implement `ai_engine.py` calling LiteLLM; add `model_tier: fast|standard|frontier` to SKILL.md frontmatter

### Output quality and UX

**16. Streaming capability execution**
Long-running capabilities (rfp_section_writer, long_form_document_engine) stream partial results token-by-token to the UI via the existing SSE infrastructure. Users see the document being written live, can interrupt and redirect mid-generation.
- *Entry point*: `web_app.py` SSE already wired (`/api/notifications/stream`); capabilities yield chunks via `notification_bus`

**17. Business development CRM layer**
Relationship graph beyond the RFP tracker: contracting officers and their preferences, teaming partners and their certifications (SDVOSB, 8(a), HUBZone), incumbent relationships, contact history. Queried automatically when writing proposals for known agencies.
- *Entry point*: new `core/crm_store.py` + `/crm` Flask routes + `rfp_customer_researcher` hook

**18. Voice-first capability invocation**
Whisper transcription tool already exists (`tools/whisper_transcribe.py`). Wire it: speak a task → transcribed → routed via SkillRegistry → result streamed back as TTS audio. Hands-free operation for field staff and executives.
- *Entry point*: `web_app.py` audio upload endpoint + `whisper_transcribe` → skill router → TTS (Ollama or piper)

### Platform extensibility

**19. Capability marketplace with versioned skill packs**
Package groups of SKILL.md + capability modules as installable tarballs: `thuon install rfp-federal-v3` or `thuon install govcon-socmed-pack`. Versioned, checksummed, diff-upgradeable. Commercial capability packs as a monetization path; community packs as a moat.
- *Entry point*: new `thuon_platform/tools/skill_pack_manager.py` + CLI subcommand `thuon install <pack>`

**20. Agentic loop via MCP tool use**
Claude Desktop uses the MCP tools (already wired at `/mcp`) to compose multi-step pipelines autonomously: "Research Agency X and write a capability statement" → Claude calls `thuon__rfp_customer_researcher`, then `thuon__rfp_win_strategy_builder`, then `thuon__rfp_section_writer`, handles errors, asks clarifying questions. Fixed YAML pipelines become the floor, not the ceiling.
- *Entry point*: MCP server is live; improvement needed is richer tool schemas + streaming result chunks via MCP content blocks
